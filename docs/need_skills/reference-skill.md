## 整理规则

- 使用普通 Markdown task list：`- [x]` 表示已选，`- [ ]` 表示候选未选；VS Code checkbox 插件可直接写回文件。
- 每条清单直接包含编号、项目、Skill、描述、记录/产物路径、路径依据和源文件，不再维护单独表格。
- 已选项归入真实项目分段；例如 `skills/hai-skills/takeoff` 归入 `hai-stack`，`skills/superpowers-skills/*` 归入 `superpowers`。
- 编号分两类：已选项使用 `NEED-SKILL-001` 起；reference 候选保留 `REF-SKILL-001` 起。
- `Auto-claude-code-research-in-sleep` 只采用 `skills-codex`，不混入顶层重复目录、Claude review 或 Gemini review 版本。
